What is Vim?

Vim (Vi IMproved) is a modal text editor built into virtually every Unix system. It's been around since 1991 and remains the editor of choice for developers, sysadmins, and anyone who values speed.

Why Vim?

  • Modal Editing: Different modes for different tasks—edit, navigate, select
  • Speed: Once learned, editing becomes incredibly fast
  • Ubiquitous: you'll Available on every server ever touch
  • Extensible: Plugins, custom keybindings, infinite customization
  • Lightweight: Runs in terminal, uses minimal resources
THE VIM PHILOSOPHY: Vim is designed around the idea that repetitive tasks should be automated. Every command can be combined, repeated with the dot operator, and mapped to anything you want.

Installing Vim

$ vim --version VIM - Vi IMproved 9.0

The Modes

Vim has several modes, but you'll use these three most:

  • Normal: The default mode for navigation and commands
  • Insert: For typing and inserting text
  • Visual: For selecting text
# In Vim: i → Enter Insert mode ESC → Return to Normal mode v → Enter Visual mode : → Enter Command mode

The Core Movement Keys

In Normal mode, these four keys are your compass:

h → Move left j → Move down (looks like an arrow) k → Move up l → Move right
TIP: These keys are on the home row. Keep your fingers here and never reach for the arrow keys again.

Word Motions

w → Jump to start of next word b → Jump to start of previous word e → Jump to end of word W → Jump word (including punctuation) B → Jump back word (including punctuation)

Line and Screen Motions

0 → Jump to beginning of line $ → Jump to end of line ^ → Jump to first non-blank character g_ → Jump to last non-blank character

Screen Motions

H → Jump to top of screen (High) M → Jump to middle of screen L → Jump to bottom of screen (Low) Ctrl+d → Move down half a page Ctrl+u → Move up half a page Ctrl+f → Page forward Ctrl+b → Page backward

Entering Insert Mode

There are many ways to enter insert mode, each with a different purpose:

i → Insert before cursor I → Insert at beginning of line a → Append after cursor A → Append at end of line o → Open new line below O → Open new line above s → Substitute (delete char and insert) S → Substitute line (delete line and insert) c → Change (combines with motion) cc → Change entire line r → Replace single character R → Replace mode (overwrite)

Leaving Insert Mode

ESC → Exit to Normal mode Ctrl+[ → Exit to Normal mode (alternative) Ctrl+c → Exit to Normal mode
PRO TIP: Remap Caps Lock to Ctrl. You'll hit Ctrl-[ constantly in Vim, and it's much easier when Caps Lock is in the bottom-left corner.

// Deleting and Changing

The Delete Commands

In Vim, most commands follow a pattern: operator + motion

x → Delete character under cursor X → Delete character before cursor dw → Delete word d$ → Delete to end of line d0 → Delete to beginning of line dd → Delete entire line 2dd → Delete 2 lines (repeat with number)

The Change Operator

Change is delete + enter insert mode:

cw → Change word (delete word and insert) c$ → Change to end of line cc → Change entire line cb → Change word backward

The Dot Operator

The most powerful command in Vim. Repeats the last change:

. → Repeat last change
EXAMPLE WORKFLOW: 1. Go to a word you want to change 2. Type cw to change the word 3. Type new word 4. Press n to go to next word 5. Press . to repeat the change 6. Press n. repeatedly to change all words

// Yanking and Pasting

×

Yank (Copy) Commands

"Yank" is Vim's term for copy:

yy → Yank entire line yw → Yank word y$ → Yank to end of line y0 → Yank to beginning of line p → Paste after cursor P → Paste before cursor

Cut and Paste

In Vim, delete is actually cut (stored in registers):

dd → Cut entire line (can paste with p) x → Cut character (can paste with p) d → Any delete is also a cut

Registers

Vim has 36 registers for storing text:

"ayy → Yank line to register a "ap → Paste from register a "bdd → Cut line to register b "0p → Paste from yank register (0) :reg → View all registers
THE BLACK HOLE REGISTER: Use "_dd to delete a line without saving it to any register. Useful when you want to cut something but not overwrite what you already copied.

// Search and Replace

×

Search Within File

/pattern → Search forward for pattern ?pattern → Search backward for pattern n → Next match (in search direction) N → Previous match (opposite direction) * → Search for word under cursor # → Search backward for word under cursor

Find and Replace

Use the :substitute command:

:s/old/new → Replace first occurrence in line :s/old/new/g → Replace all in line :%s/old/new/g → Replace all in file :%s/old/new/gc → Replace all, confirm each :s/old/new/gI → Replace all, case insensitive

Range Substitutions

:5,12s/old/new/g → Replace in lines 5-12 :.,+5s/old/new/g → Replace in current line + 5 more :'a,'bs/old/new/g → Replace between marks a and b
VERY MAGIC MODE: Use \v for regex: :%s/\v<(\w+)\gt;/\1/g removes duplicate words. Or enable with :set magic

Visual Mode Types

v → Character-wise visual mode V → Line-wise visual mode Ctrl+v → Block-wise visual mode gv → Reselect last visual selection

Visual Mode Commands

Once you've selected text, apply operators:

d → Delete selection c → Change selection y → Yank selection > → Indent right < → Indent left ~ → Toggle case U → Uppercase u → Lowercase

Block Visual Mode

Ctrl+v lets you edit columns:

EXAMPLE: Comment out multiple lines: 1. Ctrl+v to enter block mode 2. Select column with j/k 3. Shift+i to insert 4. Type // 5. ESC to apply to all lines

// Macros

×

Recording Macros

Macros record and playback sequences of commands:

qa → Start recording macro to register a q → Stop recording @a → Play back macro from register a @@ → Repeat last macro 10@a → Play macro 10 times

Practical Macro Example

Add brackets around each word on a line:

PROBLEM: Convert "foo bar baz" to "[foo] [bar] [baz]"
# Recording: qa → Start recording to a i[ → Insert [ before word e → Jump to end of word a] → Append ] after word ESC → Exit insert w → Move to next word q → Stop recording 2@q → Run macro 2 more times

Editing Macros

:reg a → View contents of register a "ap → Paste register a (to edit) "ay$ → Yank edited text back to register a

Buffers

A buffer is a file loaded into memory:

:e filename → Edit file (opens in current window) :badd file → Add file to buffer list :ls → List all buffers :bp → Previous buffer :bn → Next buffer :bfirst → First buffer :blast → Last buffer :bd 3 → Delete buffer 3 :b 3 → Jump to buffer 3

Multiple Windows

A window is a viewport into a buffer:

:sp → Split horizontally :vsp → Split vertically Ctrl+ws → Split window Ctrl+wv → Split window vertically Ctrl+ww → Cycle windows Ctrl+wh → Move to window left Ctrl+wj → Move to window below Ctrl+wk → Move to window above Ctrl+wl → Move to window right :qa → Quit all windows :wqa → Write and quit all

// Splits and Tabs

×

Splits in Detail

:sp file → Split and open file :vsp file → Vertical split and open Ctrl+w_ → Maximize current window Ctrl+w= → Equalize windows 10 Ctrl+w+ → Increase height by 10 10 Ctrl+w- → Decrease height by 10 :resize 50 → Set window height :vertical resize 80 → Set window width

Tabs

Tabs are collections of windows:

:tabnew → New tab :tabe file → Open file in new tab gt → Next tab gT → Previous tab 1gt → Go to tab 1 :tabclose → Close tab :tabonly → Close other tabs :tabmove 0 → Move tab to position 0
TIP: Use tabs for different projects, splits for working on multiple files in the same project. Learn the movement commands—they're faster than reaching for the mouse.

Text Objects

Text objects let you operate on blocks of text:

iw → Inner word (excluding whitespace) aw → A word (including whitespace) i( → Inner parentheses a( → A parentheses (including parens) i[ → Inner brackets i{ → Inner braces i" → Inner quotes i' → Inner single quotes it → Inner tag (HTML/XML)
COMBINED WITH OPERATORS: ci( = change inside parentheses da" = delete a quoted string (including quotes) yit = yank inner tag

Folds

zf → Create fold (visual mode) zF → Create fold (line-wise) zo → Open fold zc → Close fold za → Toggle fold zR → Open all folds zM → Close all folds zr → Reduce folding zm → More folding

Jumps

Ctrl+o → Go back (jump list) Ctrl+i → Go forward (jump list) Ctrl+] → Jump to tag (e.g., function definition) Ctrl+t → Pop tag (go back) g; → Go to last edit location g, → Go forward in change list :jumps → Show jump list

The Vimrc File

Your .vimrc configures Vim to your taste:

$ vim ~/.vimrc

Essential Settings

" Basic settings set nocompatible → Disable Vi compatibility syntax on → Enable syntax highlighting set number → Show line numbers set relativenumber → Relative line numbers set ai → Auto indent set si → Smart indent set tabstop=4 → Tab width set expandtab → Use spaces instead of tabs set autoindent → Preserve indentation set hlsearch → Highlight search results set incsearch → Incremental search set ignorecase → Case insensitive search set smartcase → Case sensitive if uppercase set wrap → Wrap long lines set cursorline → Highlight current line

Key Mappings

" Remap leader key let mapleader = "," " Faster escape inoremap jk <ESC> inoremap kj <ESC> " Better window navigation nnoremap <C-h> <C-w>h nnoremap <C-j> <C-w>j nnoremap <C-k> <C-w>k nnoremap <C-l> <C-w>l

Plugins

" Install plugin manager curl -fLo ~/.vim/autoload/plug.vim --create-dirs \ https://raw.githubusercontent.com/junegunn/vim-plug/master/plug.vim
" In .vimrc: call plug#begin('~/.vim/plugged') Plug 'tpope/vim-surround' Plug 'junegunn/fzf' Plug 'scrooloose/nerdtree' call plug#end()
