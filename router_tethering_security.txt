Question: Does adding a DD-WRT router between a GrapheneOS phone and Linux computer for tethering improve security over direct USB tethering?

Answer:

Direct tethering (phone → laptop via USB) is more secure.

Why:
- Fewer devices in the chain = smaller attack surface
- USB is a controlled, wired interface
- DD-WRT adds another attack vector (router firmware vulnerabilities)
- More complexity = more potential issues

What the router adds:
- No real security benefit for this use case
- Just adds another potential vulnerability
- WiFi exposure (even if laptop uses ethernet, router broadcasts)

The phone is already your security gateway. Adding a router between phone and laptop doesn't strengthen that - it just extends the attack surface.

Use cases where router makes sense:
- Multiple devices need internet
- You want to isolate other devices from your laptop
- You need better WiFi range

For security: Stick with direct USB tethering. It's simpler and more secure.

One minor note: USB tethering also keeps your phone's cellular connection more private since there's no WiFi broadcast from the phone.
