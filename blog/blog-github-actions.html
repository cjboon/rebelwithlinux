REBEL WITH LINUX

GitHub Actions

March 2026 • DevOps • 15 min read

Push code. Tests run. Deploy happens. No manual work. That's CI/CD with GitHub Actions.

What is CI/CD?

  • CI (Continuous Integration) - Automatically test your code on every push
  • CD (Continuous Deployment) - Automatically deploy when tests pass

Your First Workflow

Create .github/workflows/test.yml in your repo:

name: Run Tests

on: [push, pull_request]

jobs:
  test:
    runs-on: ubuntu-latest
    
    steps:
      - uses: actions/checkout@v4
      
      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: '20'
      
      - name: Install dependencies
        run: npm ci
      
      - name: Run tests
        run: npm test

How It Works

  • on - When to trigger (push, PR, schedule)
  • jobs - What to run
  • steps - Individual actions
  • runs-on - Which machine

Deploy on Success

jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      # ... run tests ...
      
  deploy:
    needs: test
    if: github.ref == 'refs/heads/main'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Deploy to server
        run: |
          echo "Deploying to production..."
          # Add your deploy commands

needs: test means "only run after test job succeeds".

Common Use Cases

  • Run test suites automatically
  • Lint code (check style)
  • Build Docker images
  • Deploy to cloud servers
  • Send notifications (Slack, Discord)
  • Schedule nightly builds

Matrix Testing

Test against multiple versions simultaneously:

jobs:
  test:
    runs-on: ubuntu-latest
    strategy:
      matrix:
        node: [16, 18, 20, 22]
    steps:
      - uses: actions/setup-node@v4
        with:
          node-version: ${{ matrix.node }}
      - run: npm test

Now your code is tested on Node 16, 18, 20, and 22 in parallel.
