JavaScript Basics

March 2026 • JavaScript • 20 min read

So you want to learn JavaScript. Cool. It's the language that makes websites do stuff—dropdown menus, animations, fetching data without reloading pages. Pretty much every website uses it.

Don't worry if you've never coded before. We'll take it slow.

What's JavaScript?

HTML builds the structure. CSS makes it pretty. JavaScript makes it do things.

When you click a button and something happens—that's JavaScript. When a form validates your email before submitting—that's JavaScript. When Google Maps loads new data as you scroll—that's JavaScript too.

Variables: Where You Store Stuff

Think of variables as labeled boxes. You put something in, give it a name, and can grab it later.

let name = "cjboon";
const PI = 3.14159;
var oldWay = "don't use var";

Quick breakdown:

  • let - use this. It's a box you can change later.
  • const - use this when you don't want the value to change. Like PI.
  • var - the old way. Just don't use it.

Data Types: What Goes In The Boxes

let text = "hello";      // string - just text
let num = 42;            // number - self explanatory
let bool = true;         // boolean - true or false
let arr = [1, 2, 3];    // array - a list of things
let obj = {a: 1};       // object - key-value pairs

Arrays are great for lists. Objects are great for describing things with properties.

Functions: Reusable Recipes

Functions are like recipes. You write the instructions once, then use them whenever you need.

// A named function
function greet(name) {
    return "Hello, " + name;
}

// Arrow function - shorter syntax
const add = (a, b) => a + b;

Making Decisions: If Statements

if (age >= 18) {
    console.log("You can vote!");
} else {
    console.log("Not yet, kiddo.");
}

The computer checks the condition and picks a path. Simple as that.

Loops: Doing Things Repeatedly

Don't repeat yourself. Loops let you run the same code on every item in a list.

// Classic for loop
for (let i = 0; i < 5; i++) {
    console.log(i);
}

// For each item in a list
const fruits = ["apple", "banana", "orange"];
fruits.forEach(fruit => console.log(fruit));

Interacting With The Page

This is where it gets fun. You can grab HTML elements and mess with them.

// Find a button
const btn = document.querySelector('#myButton');

// When someone clicks it
btn.addEventListener('click', () => {
    alert('You clicked it!');
});

// Change text on the page
document.getElementById('output').textContent = 'New text';

Getting Data From Elsewhere: Fetch

Want to get data from an API? That's where fetch comes in.

// Old way with .then()
fetch('https://api.example.com/data')
    .then(res => res.json())
    .then(data => console.log(data));

// New way with async/await - cleaner
async function getData() {
    const res = await fetch('https://api.example.com/data');
    const data = await res.json();
    console.log(data);
}

The async/await version reads almost like English. Much nicer.

That's It (For Now)

You now know enough JavaScript to build actual things. Try making a simple button that changes the page content. Experiment. Break stuff. That's how you learn.
