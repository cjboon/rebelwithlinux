Docker Compose

March 2026 • DevOps • 15 min read

Running one container is easy. Running a dozen that need to talk to each other? That's where Docker Compose shines.

What is Docker Compose?

Compose lets you define a multi-container application in a single YAML file. Then start everything with one command.

Installing

sudo apt install docker-compose
# or
sudo curl -L "https://github.com/docker/compose/releases/download/v2.24.0/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

A Basic docker-compose.yml

version: '3.8'

services:
  web:
    image: nginx
    ports:
      - "80:80"
    volumes:
      - ./html:/usr/share/nginx/html

The Power: Multi-Container

Here's a real WordPress setup:

version: '3.8'

services:
  db:
    image: mysql:8
    environment:
      MYSQL_ROOT_PASSWORD: secret
      MYSQL_DATABASE: wordpress
      MYSQL_USER: wp
      MYSQL_PASSWORD: wppass
    volumes:
      - db_data:/var/lib/mysql

  wordpress:
    image: wordpress:latest
    depends_on:
      - db
    ports:
      - "8080:80"
    environment:
      WORDPRESS_DB_HOST: db
      WORDPRESS_DB_USER: wp
      WORDPRESS_DB_PASSWORD: wppass
      WORDPRESS_DB_NAME: wordpress
    volumes:
      - ./wp-content:/var/www/html/wp-content

volumes:
  db_data:

Key Commands

# Start everything
docker-compose up -d

# Stop everything
docker-compose down

# View logs
docker-compose logs -f

# Rebuild (after changes)
docker-compose up -d --build

# Scale a service
docker-compose up -d --scale web=3

Why This Matters

  • One command to start your entire stack
  • Services can talk to each other by name
  • Environment variables make it portable
  • Version control your infrastructure

Your dev environment becomes reproducible. New developer joins? They run docker-compose up and have a working system in minutes.
