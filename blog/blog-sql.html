SQL Basics

March 2026 • SQL • 15 min read

SQL (Structured Query Language) is how you talk to databases. Every app needs data. SQL is how you get it.

What is a Database?

A database is a structured collection of data. Relational databases (like MySQL, PostgreSQL) organize data into tables with rows and columns.

Creating a Database

CREATE DATABASE myapp;
USE myapp;

Creating a Table

CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    name VARCHAR(100) NOT NULL,
    email VARCHAR(100) UNIQUE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

INSERT - Adding Data

INSERT INTO users (name, email) VALUES ("cjboon", "test@test.com");
INSERT INTO users (name, email) VALUES ("alice", "alice@test.com");

SELECT - Reading Data

-- All users
SELECT * FROM users;

-- Specific columns
SELECT name, email FROM users;

-- With condition
SELECT * FROM users WHERE id = 1;

UPDATE - Modifying Data

UPDATE users SET email = "new@test.com" WHERE id = 1;

DELETE - Removing Data

DELETE FROM users WHERE id = 1;

JOINs - Combining Tables

-- Users and their posts
SELECT users.name, posts.title
FROM users
JOIN posts ON users.id = posts.user_id;

Types of JOINs

  • INNER - Only matching rows
  • LEFT - All from left table, matching from right
  • RIGHT - All from right table, matching from left

ORDER BY & LIMIT

-- Newest users first
SELECT * FROM users ORDER BY created_at DESC LIMIT 10;

Aggregate Functions

SELECT COUNT(*) FROM users;
SELECT SUM(amount) FROM orders;
SELECT AVG(age) FROM users;
SELECT MAX(price) FROM products;

GROUP BY

-- Count posts per user
SELECT user_id, COUNT(*) as post_count
FROM posts
GROUP BY user_id;

You can now query any database!
